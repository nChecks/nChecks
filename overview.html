NChecks is a library giving the power of Java to monitoring scripts written in Ruby.

Monitoring scripts

Monitoring scripts must be easily customizable or created, so they are not part of the jar. You must download the "base" files here (checkout the latest release matching your NChecks {major,minor} version) and:

  • Put the ruby files in a directory accessible by the VM and initialize NChecksJRuby.startJRubyCage(scriptPath, etcdir) with the right scriptPath argument,
  • Put the xml definition were it is accessible to the world.

Monitoring executables

Some checks can't be done directly from the VM. For instance, ICMP requires a setuid executable to work correctly. So you must:

  • Build the Sysmo portable ping,
  • Put it in a directory which will be given as argument to CheckIcmp.setPping(ppingExecutable)
  • Put the xml definition were it is accessible to the world.

An working NChecks example is the NChecksErlang class.
